Monophylizer

The monophylizer is a Perl script written by Rutger Vos, made available through a webservice that is hosted by Naturalis Biodiversity Center. The monophylizer analyses trees of unlimited size for monophyletic, molyphyletic and paraphyletic 'species', recognized from taxon names. The source code is available at: https://github.com/naturalis/monophylizer.

Tree file: ▶
▶ Treat square brackets as normal text:
Normally, text inside square brackets ([]) is filtered out of newick and nexus tree descriptions before the tree structure is interpreted. However, some BOLD records incorporate these brackets for other reasons, including cases where they are not balanced (i.e. there is an opening '[' but not a closing ']'). This can cause problems when interpreting the tree, unless the square brackets are treated as normal text.
▶ Treat quotes as normal text:
Normally, single quotes '' are used so that special characters can be used in species names in newick tree descriptions. For example, to use parentheses in a name such as Homo sapiens (Linnaeus) the whole name needs to be quoted, because parentheses otherwise are interpreted as a node in a tree. BOLD doesn't respect this convention: it uses quotes in things such as Cote d'Azur. Therefore, quotes should probably be treated as normal text.
▶ Tree file format:
The default output file format for BOLD is newick. This web service can also operate on other file formats.
▶ Record separator:
The labels of the tips of trees that BOLD exports consist of a taxon name followed by an arbitrary number of metadata fields (specimen and sequence identifiers, localities, etc.). By default, the separator between these fields is the '|' symbol.
▶ Keep unescaped whitespace:
The newick standard recommends that whitespace in species names is escaped, either by replacing it with underscores ('_') or by putting the entire species name between quotes. This allows some programs to generate pretty indentation to show the 'depth' of a taxon within the tree string. BOLD does not follow this convention: whitespace occurs in unescaped form only to mean true separation between words, so normally this needs to be kept.
▶ Interpret subspecific epithets:
By default, this service assumes that any name parts following the first two, if available, are to be ignored. This means that subspecies (Foo bar vulgaris) are treated as additional instances of the containing species (Foo bar) and that polyphyly or paraphyly is therefore not assessed separately for subspecies.
▶ Additional metadata:
Here you can provide a tab-separated spreadsheet file whose first column contains species names for which you have additional metadata not included in the tree description. In the output report, these metadata are cross-referenced with the species names in the tree and included as additional columns in the table.

Suggested workflow

  1. Select the tree file you wish to analyse by clicking the file chooser.
  2. Click on the purple triangle to expand the option menu or show additional information on options. Set the correct parameters for your tree file, default settings will work for trees generated by BOLD.
  3. Click the 'Assess' button
  4. From the results page, you may copy paste everything (CTRL+A/CMD+A) into a spreadsheet (use Paste Special > only values, in Microsoft Excel).
  5. Use your browser's back button to analyse another tree file.

Q&A section (please read before reaching out for help)

Question: What does this error mean?
Software error: Can't call method "visit_depth_first" on an undefined value at /usr/lib/cgi-bin/monophylizer.pl line 254, <GEN0> line 1.
Answer: Your tree file is missing a semicolon at the end, which is a mandatory for a newick style tree format file to indicate the end of a tree line. This is a known issue with trees generated by BOLD. Open your tree file in a text editor and add a semicolon to the end prior to analysing the tree.
Question: Will the monophylizer only work on BOLD trees?
Answer: No, the script will work on a wide variety of trees, select the format of your tree file format accordingly (use the arrow to expand the menu). Just realize that the script will use the first section of the taxon name, up to the first record separator, to identify conspecific (or congeneric) taxa. It expects that the taxon name is composed of a genus name, followed by a whitespace, and then a species name: the default BOLD output. If a subspecies name is also added, separated by another whitespace, it can either ignore this or interpret this through the option menu. If there are no whitespaces at all, the script will still function but will be unable to recognize subspecies. Effectively the whole species name will then be interpreted as a genus name.
Question: Why are there single quote signs before each taxon name in the output?
Answer: BOLD does not filter taxon names for the presence of single quotes, for example when a province name like Cote D'azur is included. The single quote in Newick format indicates the ending of a taxon name, and would crash the script if present inside a taxon name. Therefore the option 'Treat quotes as normal text:' has been added. By ticking this option the script will not crash when a single quote is present inside a taxon name, but has the side effect that it will show that one is added before and at the ending of each taxon name. Only disable this option if you are certain that there are no single quotes in the taxon names. Also, keep in mind that in any additional metadata the taxon names will need to begin with a single quote as well to match the taxon names on the result page.
